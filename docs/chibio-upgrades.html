Chi.Bio — Platform-Modernisation Upgrades

Low-hanging modernisation of an exceptionally well-engineered platform — the base compute, connectivity, actuation, thermal range, and software architecture — each proposal anchored to what Chi.Bio actually ships today.

2026-07-10 Baseline: Steel et al. 2020 + tech-specs + BOM13 + ChiBio OS A design note, not an endorsement of any part number
Read this first — one correction

Chi.Bio already uses the ams AS7341 11-channel spectral sensor — the BOM board is literally SensorBreakoutAS7341, and the OS instantiates AS7341 at I²C 0x39, reading eight bands plus Clear/NIR. So "upgrade the 8-band array to an AS7341" is not an upgrade; it is the current part. §4 covers what a real optical upgrade would be. This note is the base-platform axis; the companion two-instruments Part 1 covers new sensing modalities (DO/pH optodes, dispersive spectrometry, atmosphere) and this note defers to it on optics.

01 Control computer & connectivity — highest impact, not lowest effort

Effort: highAdopt · budget HAL rewrite

BeagleBone Black → Raspberry Pi (Zero 2 W, 4, or 5)

The BBB is single-core with no onboard wireless; a Pi Zero 2 W is quad-core with WiFi + BT, and Pi 4/5 add compute for logging, deconvolution, and web-UI responsiveness — plus broader supply and community. But Chi.Bio's I/O layer is BeagleBone-specific (Adafruit_BBIO, the PRU/PWM/GPIO mapping, the cape form factor). A Pi port means rewriting the hardware abstraction layer and respinning the interface board — the biggest single change here, worth it but not a swap.

Effort: lowAdopt

micro-USB → USB-C on the reactor link

micro-USB ports are the classic mechanical failure point under lab handling; USB-C is more durable and unlocks better power-delivery options. The link is data-only (power is already separate), so this is a connector + cable change.

02 Liquid handling & pumping

Effort: mediumAdopt selectively

Brushed-DC peristaltic → stepper-driven peristaltic

Chi.Bio meters low flow by PWM-duty-cycling brushed-DC pumps, which pulses the flow. Stepper heads give smooth, pulsation-free micro-metering of media and inducers, with repeatable volume per step. New drivers + firmware and more cost per channel — adopt where metering precision matters; keep DC for coarse waste.

Effort: low–medAdopt for longevity

Brushed → brushless DC (if staying DC)

Brushless motors remove the brush-wear failure mode that kills pumps mid-run on multi-week continuous cultures.

03 Thermal management

Effort: mediumAdopt (variant)

Resistive heater → Peltier (thermoelectric) module

Chi.Bio's 3 W resistive plate with passive cooling can only hold above ambient. A Peltier heats and actively cools, enabling psychrophilic organisms and precise cold-shock / cold-inducible optogenetic protocols. Costs a heatsink/fan, an H-bridge driver, and more power budget — and watch condensation on the optical path. A real capability gain, not just a spec bump.

04 Optics — read the correction first

In-family: lowStep-change: high

The premise correction, then two real options

Chi.Bio already has the AS7341 (11 channels), so "upgrade to an AS7341" is a no-op. Two real options exist: in-family, the ams AS7343 (14-channel) is a near-drop-in for a few more bands and marginally cleaner GFP/YFP/mCherry deconvolution (low effort, modest payoff); step-change, a dispersive micro-spectrometer (Hamamatsu C12880MA) replaces fixed filter bands with a continuous 340–850 nm spectrum — the higher-value optical upgrade, already specced in two-instruments Part 1, so defer there.

Effort: highVariant only

Discrete, standard-footprint excitation LEDs

Chi.Bio's excitation is one multi-die seven-colour emitter behind focusing optics — powerful, but not a set of swappable discrete LEDs. Re-laying the optical PCB with discrete Star/MCPCB or 3535 footprints would let users populate their own excitation wavelengths (e.g. a specific optogenetic driver line) from off-the-shelf parts. It trades the integrated emitter's focusing and co-alignment for modularity — a redesign, not a swap. Worthwhile for an optogenetics-focused variant; know the trade.

05 Software & integration architecture

Effort: low–medAdopt

Containerise the backend (Docker)

Chi.Bio installs natively on the BeagleBone via shell scripts, with no Dockerfile; Python-dependency drift is a real setup pain. A container makes install one command and pins the stack. Note the current armv7/32-bit target — a Pi move (§1) eases this.

Effort: mediumAdopt · biggest integration win

Add a REST or MQTT layer

The Flask UI exposes JSON endpoints for its own front-end but no documented external API; automation today means editing the Python app. A small REST/MQTT surface makes Chi.Bio composable with cloud-lab schedulers and liquid-handling robots (Opentrons, etc.).

07 Already present — do not propose these

Verified against the BOM and control code, so an "upgrade" list should not include them:

  • ams AS7341 11-channel spectral sensor — already the baseline optical part.
  • Independent per-pump speed control — already done via PWM (the issue is pulsation, not lack of control).
  • Per-colour excitation control — the seven-colour emitter already drives each band independently.
  • Python + web UI — already the stack; the gap is packaging and an external API, not the UI.
